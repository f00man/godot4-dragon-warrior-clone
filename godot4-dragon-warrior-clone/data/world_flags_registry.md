# World Flags Registry
# godot4-dragon-warrior-clone
#
# Every world flag that gets set anywhere in the game must be documented here.
# A poorly named or undocumented flag is a hard-to-find bug waiting to happen.
#
# Columns:
#   Flag Key          — the exact string key used in GameState.world_flags
#   Type              — bool | int | string
#   Set By            — which event id, system, or function sets this flag
#   Cleared By        — what clears or resets the flag (if applicable)
#   Meaning           — plain-English description of what "truthy" means for this flag
#
# Naming rules:
#   Good: "rescued_village_of_kale", "sided_with_merchant_guild"
#   Bad:  "flag_1", "quest_done", "choice_a"
#
# Completion flags (format: "event_<id>_done") are set automatically by
# EventManager.apply_outcome() when set_completion_flag is true in an outcome.
# They are listed in the Completion Flags section below.

---

## Player Exploration Flags

| Flag Key           | Type | Set By             | Cleared By | Meaning                                          |
|--------------------|------|--------------------|------------|--------------------------------------------------|
| found_bandit_camp  | bool | player_exploration | —          | Player has discovered the bandit camp near Riverkeep and the event is now eligible to trigger |

---

## Event Outcome Flags

Flags set as direct consequences of player choices inside events.

| Flag Key              | Type | Set By                 | Cleared By | Meaning                                                            |
|-----------------------|------|------------------------|------------|--------------------------------------------------------------------|
| resolved_bandit_camp  | bool | bandit_camp_choice     | —          | The bandit camp situation near Riverkeep has been resolved (by any means — attack or diplomacy) |
| bandits_allied        | bool | bandit_camp_choice     | —          | Player negotiated a truce; the bandits are now allied with the player rather than hostile |

---

## Event Completion Flags

Auto-generated by EventManager.apply_outcome() when set_completion_flag is true.
Format: "event_<event_id>_done". Checked by EventManager.is_event_completed().
These flags live in world_flags so SaveManager persists them with no special handling.

| Flag Key                       | Type | Set By                           | Meaning                                                         |
|--------------------------------|------|----------------------------------|-----------------------------------------------------------------|
| event_bandit_camp_choice_done  | bool | EventManager (bandit_camp_choice) | The bandit camp event has been presented and the player has made their choice; do not present it again |
